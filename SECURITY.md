# Politique de sécurité

## Versions prises en charge

Avant la premiére release stable, seule la branche de développement actuelle est prise en charge.

## Signaler une vulnérabilité

N'ouvrez pas d'issue publique contenant un secret exploitable ou les détails d'une vulnérabilité non corrigée.

Signalez le probléme de maniére privée au mainteneur du projet, avec :

- une description concise ;
- les versions concernées ;
- des étapes de reproduction minimales ;
- l'impact estimé ;
- une proposition de correction, si disponible.

Le projet accuse réception, évalue l'impact et coordonne une correction avant toute divulgation publique.

## Portée

Watchglass est un outil local d'hygiéne de sécurité. Il ne réalise ni scan offensif, ni exploitation, ni appel réseau.
